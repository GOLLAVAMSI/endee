## 1. PROJECT ARCHITECTURE Overview
CyberSentinel AI is a Threat Intelligence platform built on a modern Retrieval-Augmented Generation (RAG) architecture.
The goal of the system is to take user cybersecurity queries (e.g., "What is SQL Injection?"), retrieve the most relevant 
threat intelligence data from a Vector Database, and use an advanced Large Language Model (LLM) to generate a detailed, 
context-aware response.

### 1.1 The Flow of the Application:
- 1. User Interface (UI): The user interacts with a Streamlit-based web UI. 
- 2. Data Ingestion: Upon startup, the app loads threat data from `data/cyber_data.json` and converts it into mathematical 
     vector embeddings using the `sentence-transformers` library (Model: all-MiniLM-L6-v2). 
- 3. Vector Database: These embeddings are stored in a Local Vector Store (falling back safely if Endee DB is unavailable).
- 4. Context Retrieval (RAG): When the user asks a question, the system converts the question into an embedding and runs 
     a "similarity search" in the Vector Store to find the top matching intelligence reports.
- 5. Memory Management: The system checks previous interactions to provide conversational context.
- 6. LLM Generation: The retrieved facts and the user's question are sent to a Groq-powered LLM (Llama 3 70B), 
     which synthesizes a highly accurate, structured Threat Intelligence Report.

FILE: `modules/local_vector_store.py`
--------------------------------------------------
This file handles the Vector Database fallback logic if Endee is not deployed.
- `import numpy as np`, `import uuid`, `from sentence_transformers import SentenceTransformer`: Core ML libraries.
- `class LocalNumPyVectorStore:` : Defines our local vector database.
- `def __init__(self, use_mock=False):` : Loads the `all-MiniLM-L6-v2` embedding model.
- `def _get_embedding(self, text):` : Converts a string text into a dense vector (array of numbers) representing its semantic meaning.
- `def insert(self, collection_name, vector=None, metadata=None, ...)` : Adds a document and its embedding to memory.
- `def search(self, collection_name, query_vector=None, ...)` : Computes "Cosine Similarity" (the math that finds similar meanings) between the user's question embedding and all stored document embeddings. Returns the top `k` closest matches.

--------------------------------------------------
FILE: `modules/rag_pipeline.py`
--------------------------------------------------
This file defines the Intelligence logic joining the DB and the LLM.
- `from openai import OpenAI` : We use the OpenAI python library, but configured to point to Groq's super-fast inferencing API.
- `def __init__(self, vector_client, ...)` : Connects to the LLM and the Vector DB.
- `def generate_response(self, query, top_k=3, memory_context=""):`
  - 1. Converts the user `query` into a vector embedding.
  - 2. Uses `self.vector_client.search()` to find `top_k` documents in the Vector DB matching the query.
  - 3. Extracts the text content (`point['payload']['content']`) from the DB results to form the `context`.
  - 4. Merges the Context, Memory, and User Query into a structured System Prompt.
  - 5. Sends the customized prompt to the Groq LLM model (`llama3-8b-8192` or `llama3-70b-8192`).
  - 6. Returns the final text string and citations.

--------------------------------------------------
FILE: `modules/data_loader.py`
--------------------------------------------------
This script initializes the Vector Database.
- `def __init__(self):` : Tries to connect to the Endee Vector Database using an API. If it fails, perfectly switches over to `LocalNumPyVectorStore()`.
- `def initialize_index(self):` : Checks if the "cyber_knowledge" and "chat_memory" tables exist in the DB.
- `def ingest_data(self, file_path="data/cyber_data.json"):` 
  - Loads the hardcoded JSON threat data.
  - For each cybersecurity entry, it generates an embedding vector.
  - Pushes the text and vector representation into the Vector Store to be searched later.

--------------------------------------------------
FILE: `modules/memory.py`
--------------------------------------------------
Controls conversational thread retention.
- `def store_interaction(self, query, response):` : Saves a user question and AI answer pair as a string, creates an embedding for it, and stores it in the `chat_memory` table.
- `def get_relevant_history(self, current_query, top_k=2):` : When a new question comes in, it searches the `chat_memory` table to see if old questions are similar and returns them to guide the AI.
- `def get_recent_queries(self, limit=5):` : Retrieves chronological recent questions to display in the UI sidebar.

## 3. SUMMARY
========================================================================
CyberSentinel AI is a masterclass in modern Enterprise AI engineering:
1. It separates the UI (app.py) from the core Logic (modules).
2. It uses RAG to avoid "AI Hallucinations" by strictly feeding contextual cybersecurity facts into the LLM context limits.
3. It intelligently embeds inputs to allow semantic search capabilities rather than keyword searches.
4. It utilizes a highly optimized LLM inference endpoint (Groq) to provide nearly instant feedback.
5. Employs advanced UI/UX to deliver a premium user experience akin to custom industry software.
